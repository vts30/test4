================================================================================
  perf-lib-v2 — Database Design
================================================================================

OVERVIEW
--------
Two-table design: one row per test run (test_runs) and one row per HTTP
observation (observations). Everything is written in a single transaction
inside AfterAll — no partial writes, no status column needed.


TABLES
------

test_runs
  id              UUID         PRIMARY KEY  — auto-generated, used as FK in observations
  build_id        TEXT                      — Jenkins BUILD_NUMBER (env: BUILD_NUMBER)
  git_repo        TEXT                      — URL of the application repo (env: GIT_URL)
  git_hash        TEXT                      — commit SHA of the application (env: GIT_COMMIT)
  git_branch      TEXT                      — branch of the application (env: GIT_BRANCH)
  test_git_repo   TEXT                      — URL of the test repo (env: TEST_GIT_REPO)
  test_git_hash   TEXT                      — commit SHA of the test repo (env: TEST_GIT_HASH)
  test_git_branch TEXT                      — branch of the test repo (env: TEST_GIT_BRANCH)
  environment     TEXT         NOT NULL     — e.g. 'dev', 'staging', 'prod' (env: PERF_ENV)
  test_suite      TEXT         NOT NULL     — e.g. 'Anmelden' (env: PERF_TEST_SUITE)
  sprint          TEXT                      — sprint identifier (env: SPRINT)
  started_at      TIMESTAMPTZ  NOT NULL     — when BeforeAll ran
  finished_at     TIMESTAMPTZ  NOT NULL     — when AfterAll ran (= NOW() at commit time)
  tags            JSONB        NOT NULL DEFAULT '{}'
                                            — e.g. {"golden": "true"} to mark a reference run
  created_at      TIMESTAMPTZ  NOT NULL DEFAULT now()

  CONSTRAINT uq_run_identity UNIQUE (build_id, test_suite, environment)
    — prevents duplicate runs from the same Jenkins build

observations  (PARTITION BY HASH (run_id), 8 partitions: observations_p0 … p7)
  id           BIGINT  GENERATED ALWAYS AS IDENTITY
  run_id       UUID    NOT NULL REFERENCES test_runs(id) ON DELETE CASCADE
  metric_name  TEXT    NOT NULL   — scenario name (or usePerfContext name)
  value        DOUBLE PRECISION NOT NULL   — response time in milliseconds
  recorded_at  TIMESTAMPTZ NOT NULL        — time the HTTP response was received
  attributes   JSONB   NOT NULL DEFAULT '{}'
                 — {url, method, status_code, response_size_bytes, content_type, assertions}
  PRIMARY KEY (run_id, id)   — composite PK required for hash partitioning


INDEXES
-------
idx_observations_metric    ON observations(metric_name)
idx_observations_recorded  ON observations(recorded_at DESC)
idx_test_runs_environment  ON test_runs(environment)
idx_test_runs_started      ON test_runs(started_at DESC)
idx_test_runs_golden       ON test_runs(test_suite, environment)
                              WHERE tags->>'golden' = 'true'
  — partial unique index: only one golden run allowed per suite+environment


PARTITIONING
------------
observations is partitioned by HASH(run_id) into 8 sub-tables.
PostgreSQL routes each INSERT automatically based on the run_id value.
Benefits:
  - parallel query execution across partitions
  - faster pruning when filtering by run_id
  - easier archiving (drop old partitions by date if needed later)


TRANSACTION FLOW
----------------
BeforeAll  : browser + RunManager created, no DB write
Before     : recorder + context set up per scenario, no DB write
After      : HTTP records buffered in memory (queue.enqueue), no DB write
AfterAll   :
  1. queue.drain() — collect all buffered observations
  2. BEGIN
  3. INSERT INTO test_runs ... RETURNING id
  4. INSERT INTO observations (batch) using run_id from step 3
  5. COMMIT  (or ROLLBACK on any error)
  6. closePool(), browser.close()


ENV VARS (Jenkins / .env.satu)
-------------------------------
Required:
  PERF_ENV          target environment name
  PERF_TEST_SUITE   name of the test suite
  PG_HOST / PG_USER / PG_PASSWORD / PG_DB   database connection

Optional:
  BUILD_NUMBER      Jenkins build number → build_id
  GIT_URL           app repo URL         → git_repo
  GIT_COMMIT        app commit SHA       → git_hash
  GIT_BRANCH        app branch           → git_branch
  TEST_GIT_REPO     test repo URL        → test_git_repo
  TEST_GIT_HASH     test commit SHA      → test_git_hash
  TEST_GIT_BRANCH   test branch          → test_git_branch
  SPRINT            sprint identifier    → sprint
  PERF_VERSION      arbitrary version string (stored in context, not DB column)
  PERF_CSV_PATH     if set, write CSV instead of DB (local dev fallback)
  DATABASE_URL      full connection string (overrides individual PG_* vars)


CSV FALLBACK (local dev)
------------------------
Set PERF_CSV_PATH=/path/to/output.csv to skip the DB entirely.
Each row is a flat denormalized record containing all test_run fields
plus all observation fields.

CSV columns:
  run_id, build_id, git_repo, git_hash, git_branch, test_git_repo,
  environment, test_suite, sprint, metric_name, value, recorded_at,
  url, method, status_code, response_size_bytes, content_type, assertions


GOLDEN RUN
----------
To mark a run as the reference baseline, set tags->>'golden' = 'true'.
The partial unique index idx_test_runs_golden ensures only one golden run
exists per (test_suite, environment) combination at any time.
Use ON CONFLICT or update the existing golden tag before inserting a new one.


USEFUL QUERIES
--------------
-- Last 5 runs:
SELECT id, build_id, environment, test_suite, started_at, finished_at
FROM test_runs ORDER BY started_at DESC LIMIT 5;

-- Observation count per run:
SELECT run_id, COUNT(*) FROM observations GROUP BY run_id ORDER BY COUNT(*) DESC;

-- Average response time per metric for a run:
SELECT metric_name, ROUND(AVG(value)::numeric, 2) AS avg_ms, COUNT(*) AS hits
FROM observations
WHERE run_id = '<uuid>'
GROUP BY metric_name ORDER BY avg_ms DESC;

-- Compare a run against the golden run:
SELECT o.metric_name,
       ROUND(AVG(o.value)::numeric, 2)   AS current_avg_ms,
       ROUND(AVG(g.value)::numeric, 2)   AS golden_avg_ms,
       ROUND((AVG(o.value) - AVG(g.value))::numeric, 2) AS diff_ms
FROM observations o
JOIN observations g ON g.metric_name = o.metric_name
JOIN test_runs r  ON r.id = o.run_id
JOIN test_runs gr ON gr.id = g.run_id AND gr.tags->>'golden' = 'true'
                  AND gr.test_suite = r.test_suite
                  AND gr.environment = r.environment
WHERE o.run_id = '<uuid>'
GROUP BY o.metric_name ORDER BY diff_ms DESC;
